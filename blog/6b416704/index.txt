<h2 style="color: #C3E88D;">
                  <span style="font-size: initial; color: #F0C292; margin-right: 0.5em;">h2</span>
                  단점
                </h2>
<ul>
<li>복잡성(Complexity) : 정확히 무엇을 사용하는지 헷갈림</li>
<li>오버헤드(Overhead) : 런타임시에 동작 결정하기 때문에 연산이 많이 들어감</li>
<li>어려운디버깅(Difficulty of debugging) : 지금 내가 보는 이게 무엇인지 알기 힘듬</li>
</ul>
<h2 style="color: #C3E88D;">
                  <span style="font-size: initial; color: #F0C292; margin-right: 0.5em;">h2</span>
                  서브타입 다형성 (Subtype polymorphism)
                </h2>
<ul>
<li>부모의 인터페이스를 참조하고 사용되는 것은 자식이 <a href="/github-blog/blog/d0192ff4">오버라이딩 (overriding)</a> 한 내용을 사용하도록 하는 방법</li>
</ul>
<h2 style="color: #C3E88D;">
                  <span style="font-size: initial; color: #F0C292; margin-right: 0.5em;">h2</span>
                  매개변수 다형성 (Parametric polymorphism)
                </h2>
<ul>
<li>타입을 매개변수로 받아 새로운 타입을 되돌려주는 기능</li>
<li>타입 매개변수를 정의한 클래스 혹은 메소드는 사용할 때 매개변수에 타입에 타입을 지정하며 <mark>컴파일 시 지정한 타입에 따라 해석</mark></li>
<li>
<h3 style="color: #C792EA;">
                  <span style="font-size: initial; color: #F0C292; margin-right: 0.5em;">h3</span>
                  템플릿(Template)
                </h3>
<ul>
<li>C++ 에서 사용된 개념</li>
<li>타입 매개변수를 입력한 타입으로 치환한 코드를 생성하는 방식</li>
</ul>
</li>
<li>
<h3 style="color: #C792EA;">
                  <span style="font-size: initial; color: #F0C292; margin-right: 0.5em;">h3</span>
                  제너릭 (Generic)
                </h3>
<ul>
<li>Java와 C# 등에 도입된 개념</li>
<li>지정한 타입 매개변수에 해당하는 타입만을 사용하겠다고 약속하는 방식</li>
<li>타입 매개변수가 특정 객체를 상속할 경우 상속하는 객체의 함수는 호출할 수 있지만 그렇지 않을 경우 타입 매개변수로 지정된 객체의 멤버에는 접근 불가</li>
</ul>
</li>
</ul>
