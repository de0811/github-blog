                  REST API
                </h1>
<ul>
<li>특정한 설계 규칙을 지키는 것</li>
</ul>
<h2 style="color: #C3E88D;">
                  <span style="font-size: initial; color: #F0C292; margin-right: 0.5em;">h2</span>
                  주요 원칙
                </h2>
<ul>
<li><strong>무상태(Stateless)</strong>: 각 요청은 독립적이어야 함</li>
<li><strong>자원 중심</strong>: URL이 자원을 나타내야 함 (<code>/users/123</code>)</li>
<li><strong>HTTP 메소드 활용</strong>: GET, POST, PUT, DELETE 등을 의미에 맞게 사용</li>
<li><strong>표현</strong>: JSON, XML 등으로 데이터 표현</li>
<li><strong>계층화</strong>: 클라이언트는 서버 구조를 알 필요 없음</li>
</ul>
<h2 style="color: #C3E88D;">
                  <span style="font-size: initial; color: #F0C292; margin-right: 0.5em;">h2</span>
                  RPC(Remote Procedure Call) 스타일
                </h2>

<li><code>RESTful RPC</code> / <code>REST-like RPC</code> / <code>REST-ish</code> / <code>Pragmatic REST</code> 등으로 표현</li>

</ul>
