# 도메인 주도 설계(DDD) Domain-Driven Design
- 도메인(업무 영역) 자체를 중심에 두고 모델을 만듦
- 그 모델을 코드에 직접 반영하는 방법론
## 🚨 주의사항
## 🔍 속성 설명
- 도메인 모델: 실제 업무를 추상화한 개념 모델로, 업무 규칙과 개념을 코드로 표현합니다.
- 유비쿼터스 언어: 도메인 전문가와 개발자가 공통된 언어(용어)를 사용함으로써 모델과 코드 간에 일관성을 유지하고 오해를 줄입니다.
- 바운디드 컨텍스트: 도메인을 여러 구역(컨텍스트)으로 나누어, 각 구역에서 모델을 독립적으로 정의하고 관리합니다.
- 애그리거트: 일관성을 유지해야 하는 객체들의 그룹으로, 대표 엔티티(Root Entity)를 통해서만 데이터를 변경하도록 합니다.
- 도메인 서비스: 특정 엔티티나 값 객체에 넣기 애매한 로직을 명시적으로 분리하여 관리합니다.
- 리포지토리: 애그리거트 단위로 영속성 작업(저장, 조회 등)을 처리합니다.
	DDD 에서는 모델의 설계와 구현이 긴밀히 연결되며, 팀원 간의 협업을 통해 도메인을 깊이 이해하고 반영하는 과정이 중요합니다.
